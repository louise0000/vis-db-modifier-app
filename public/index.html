CSV Import Review

      
Upload a CSV, review fuzzy duplicate suggestions row by row, choose create / merge / skip, and resolve parent matches before writing anything.


      

        
        
        
      


      
Leave any unknown field blank. Parent and child columns may contain labels for review/matching after upload; parentId and childId columns are for explicit UUIDs. Other columns become info fields. Colour is filled automatically from the database's existing colour for that type when the CSV leaves color blank.


      

        Example CSV structure
        

          
Blank cells are fine. You do not need to type labels such as “birth:” into values; birth, death and date are separate columns.

Multiple parent or child labels can be separated with semicolons. Commas are preserved inside names and titles. Every textual relationship must be explicitly resolved or confirmed as no attachment before commit. Maximum 500 non-empty rows per batch. The importer checks both existing database records and other rows in the same CSV batch. Type colour is added automatically when color is blank or omitted.
